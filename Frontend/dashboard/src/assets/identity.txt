The Identity Control Center is the main dashboard for users to see and manage all their Shield Identities. It provides a clear overview of each synthetic identity generated by the browser extension, including its status and activity, without ever exposing the user’s real identity.

Step-by-Step Functionality

Data Source

The dashboard receives real-time data from the backend/API about all Shield Identities created for the user.

Each identity record contains:

Shield email

Virtual phone number

Associated website/service

Creation date

Status (Active / Suspicious / Burned)

Last activity timestamp

Display

The front-end renders a table or card view showing each identity’s details.

Status is color-coded:

🟢 Active → Everything is normal

🟡 Suspicious → AI flagged unusual activity

🔴 Burned → Identity has been invalidated due to a threat

Example table:

Website	Shield Email	Phone	Status	Last Activity
Netflix	netflix@shieldmail.ai
	+1xxxx	🟢 Active	2h ago
Amazon	amazon@shieldmail.ai
	+1xxxx	🟡 Suspicious	1h ago

User Interaction

Users can sort by website, creation date, or status.

Filter to see only active, suspicious, or burned identities.

Optionally click on an identity to see detailed logs or communications associated with it.

Integration with Project Workflow

Shielded Onboarding: New identities automatically appear on the dashboard as soon as the browser extension creates them.

Anomalous Detection: Status changes to “Suspicious” when the AI engine flags unusual behavior.

Autonomous Self-Healing: When an identity is burned and replaced, the dashboard updates in real-time to show the new identity, preserving the audit trail.

UI/UX Design

Clean, responsive layout.

Color-coded statuses make monitoring quick and intuitive.

Supports real-time updates, so users always see the current state of their Shield Identities.

Framework / Library

React.js → Widely used, component-based, great for dynamic dashboards.

Alternative: Vue.js (lightweight, reactive) or Angular (enterprise-level, structured).

State Management

React Context API or Redux → For managing identities and dashboard state.

Allows real-time updates when new identities are created, burned, or flagged suspicious.

UI Component Library / Styling

Tailwind CSS → Modern, utility-first styling, responsive out of the box.

Optional: Material-UI / Chakra UI / Ant Design → Ready-made tables, badges, and responsive components.

API Integration

Axios or native fetch() → To fetch data from the backend API that provides Shield Identity logs.

Supports real-time updates through polling or WebSocket for live changes.

Real-Time Updates (Optional)

Socket.IO or Pusher → For live status updates when AI flags suspicious behavior or a self-healing rotation occurs.

Data Visualization (Optional)

Recharts or Chart.js → If you want to show metrics like “number of active vs burned identities” or activity trends.

Routing / Navigation

React Router → To navigate between the Identity Control Center, detailed views, and possibly other dashboards.

Optional Enhancements

React Table → For sortable/filterable tables.

React Query / SWR → For efficient API fetching and caching.
